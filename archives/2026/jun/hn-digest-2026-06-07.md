## 14. [Speculative KV coding: losslessly compressing KV cache by up to ~4×](https://news.ycombinator.com/item?id=48400151) (↑119, 💬19)
**Article:** The article introduces "Speculative KV coding," a lossless compression method for LLM KV caches that uses a smaller, cheaper predictor model to estimate the target model's cache output. The predictor runs in parallel on both encode and decode sides, and an arithmetic coder encodes the true cache at a bitrate set by how well the predictor fits the target, achieving up to 4x compression on top of lossy FP8 compression. The key insight is that the KV cache is deterministic given the prompt and model weights, so entropy can be eliminated by modeling the forward pass.

**Comments:** Comments are mixed, with praise for the article's clarity but skepticism about the practical utility. Some commenters note that recomputing even a draft cache is still quadratic in context length, and that the approach requires decompressing the entire cache to generate a single new token, making it unsuitable for VRAM savings in practice. Others discuss potential trade-offs for very large models and batch sizes.
