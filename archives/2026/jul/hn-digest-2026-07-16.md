## 14. [GC shape stenciling in Go generics](https://news.ycombinator.com/item?id=48877151) (↑19, 💬1)
**Article:** The article explains Go's approach to implementing generics through "GC shape stenciling," which sits between full monomorphization (Rust/C++) and type erasure (Java). Go generates compiled code per "GC shape" based on type size, alignment, and pointer content, then uses dictionaries for type-specific operations. This reduces binary size compared to full monomorphization while maintaining good performance.

**Comments:** The single comment notes that other languages like Swift and C#/.NET also use dictionary-based approaches similar to Go's. Swift uses "witness tables" for memory management across different type shapes, while C# monomorphizes at JIT time for value types but shares implementations for reference types.
